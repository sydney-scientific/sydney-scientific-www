Apache Mesos

Mesos was created for resource allocation at a large-scale.

Datacentres of heterogeneous loads. Before Mesos, clusters were [commonly] statically partitioned. These 15 nodes for Postgres, these 15 for applications, these 15 for recommender systems. At most, you'd configure the latter to regress during idle time. The Mesos solution is to think of all these nodes as a single resource pool, which has many advantages for cost, data locality, and speed (the researchers invented Spark in the same project).

Mesos is horrible at small scale, and great at medium to 10,000 node scale.

Our contributions aim to solve this problem. We decouple Apache ZooKeeper, and give the Mesos deployer a choice between: ZooKeeper, etcd and consul.

Compiler Driven Development (CDD)

In application development, projects tend to be multi-tier (e.g.: database, backend, and frontend). Nowadays, this often includes multi frontends (natives [incl. mobile] apps, web app).

Something as trivial as adding a field to a model result in ~36 lines of code changes (Swift for iOS, Angular for web, Java for Android, Rust for backend).

CDD solution is straightforward: traverse the Abstract Syntax Tree (AST) in each language, translate it to our SQL-extended OpenAPI format. Enable tests, documentation, validation, routing, and auto-admin views to be generated. Yet still, always enable custom development of any file, in any language.

CI/CD

There are over 50 different CI/CD formats and platforms on the market, with ~10 commonly used. Travis CI, CircleCI, Gitlab CI, Azure pipelines, AWS CodePipeline, and Google cloudbuild; to name a few.

By developing parser/emitters for each major format—in Rust—we remove vendor lock-in. Related work: github/gh-actions-importer

What is run on, say Travis CI, is hard to replicate locally. By creating small tools in Rust, we can run the equivalent during development. Projects like Gitlab require a large amount of resources allocated. Ours should run well at 1 node. And many. Hint: see Mesos!

FreeBSD jails and Solaris zones have lost interest in the shadow of Docker. There's no reason Dockerfile can't be just another format supported, thus allowing these other environments to be targeted.
